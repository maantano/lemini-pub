사용자의 질문·쟁점과 아래 판례 후보를 비교해, 질문의 **쟁점과 직접 관련된** 판례를 관련도 높은 순서대로 골라라.

## 판정 기준 (우선순위 순)

1. **쟁점 일치** — 질문이 묻는 법리(구성요건·해석 기준·권리의무 관계·절차 요건)를 판례가 직접 다루는가.
2. **사실관계 유사성** — 판례의 사실관계가 질문 상황과 유사하거나 유형적으로 비교 가능한가.
3. **최신성** — 최근 선고일일수록 가산점 (단, 쟁점 일치가 항상 더 중요).
4. **위계** — 대법원 > 고등법원 > 지방법원. 다만 위계만 높다고 더 관련있는 건 아니다.
5. **관련성이 희박하면 배제** — 억지로 넣지 마라. 관련성이 낮은데도 '판례가 있으니까' 넣으면 오히려 답변 품질이 떨어진다.

## 원칙

- 특정 법 분야·업종을 가정하지 마라. 질문 텍스트와 판례 텍스트에서 드러난 정보만으로 판단하라.
- 판례 후보의 제목·법원·선고일·사건번호·판시사항·판결요지만 근거로 사용하라.
- 개별 판례가 **질문과 어떻게 연결되는지 설명할 수 있어야** 관련도 높음으로 판정한다.

## 출력 (반드시 JSON)

관련성 높은 순서대로 **판례 후보의 인덱스 번호**만을 JSON 배열로 나열하라. 관련성이 낮거나 불분명한 후보는 배제해도 된다.

예시:
- `[2, 0, 3]` — 2번·0번·3번 3개가 순서대로 가장 관련
- `[4]` — 1개만 관련
- `[]` — 관련 판례 전무

규칙:
- 배열 외 다른 텍스트·설명·코드펜스 출력 금지.
- 인덱스는 0부터 시작.
- 최대 {max_items}개까지 반환 (설정된 경우). 넘지 마라.
- 중복 인덱스 금지.
